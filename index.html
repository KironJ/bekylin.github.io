hadoop null does not belong to hadoop

  • 问题
  • 再次寻求解决

问题

  1. 用户在执行hive的load语句将数据导入hive中时, 会出现这个异常.
  2. sqoop语句部分存在这个问题.
  3. hive语句部分存在这个问题.

hadoop null does not belong to hadoop

这个问题困扰我很久了, 上网查, 看到CSDN上有几篇文章说是因为在主机中没有添加执行这个操作的用户. 后面我在机器中添加了相应的用户, 并且加入到Hadoop和HDFS组中. 问题并没有解决.
另外, 管理员执行同样的语句是可以正常执行, 而不会出现这个异常的.

再次寻求解决

后来查到问题是源于hive的一个bug. 链接里提供了相对应的解决办法.

  • user-does-not-belong-to-hdfs
  • HIVE-13716

安装hexo

之前有过安装hexo的经历, 但是也只仅限于安装…
这真是一个悲伤的故事.
所以,现在要, 一雪前耻 (๑•̀ㅂ•́)و✧慢慢加油, 太用力了跑不远

安装hexo的文章一搜一大把, 我这里只是简单记一下重点内容好了.

1. 安装 hexo包

npm install -g hexo-cli

2. 安装git

git原来就一直在用了, 所以就不需要重新安装.

3. 申请gitpage

在网上搜索也是一大堆, 重点记住的是git repo的名字要和github的名字一致.

3. 在hexo中配置gitpage

直接把配置文件拿出来了, 参考的时候照葫芦画瓢吧!

安装目录插件.

实现这个功能需要安装插件.

1. 找到hexo-renderer-markdown-it-plus插件

在hexo插件查找页面找到插件: hexo-renderer-markdown-it-plus.
这个插件是fork于hexo-renderer-markdown-it的.

2. 安装这个插件

需要将原来hexo自带的renderer替换掉.

3. 在文章中使用

在文章中以@[toc]形式使用, 但是我发现个现象就是标题1是不会显示的. 最目录里显示的最大的标题是标题2!

4. 不只是目录

错误

安装的过程中遇到了一点问题.

1. 执行hexo init的时候找不到一些module.

我干脆进到全局的node modules文件夹中, 找到hexo的node_modules, 把里面的都删掉又重新安装了一下就好了.

2. 执行hexo deploy的时候失败.

上网找, 发现是少了个组件.
在我的hexo的home目录执行命令npm install hexo-deployer-git --save
