安装hexo

之前有过安装hexo的经历, 但是也只仅限于安装.... 这真是一个悲伤的故事. 所以,,,现在要,, 一雪前耻 (๑•̀ㅂ•́)و✧慢慢加油, 太用力了跑不远

安装hexo的文章一搜一大把, 我这里只是简单记一下重点内容好了.

1. 安装 hexo包

npm install -g hexo-cli

2. 安装git

git原来就一直在用了, 所以就不需要重新安装.

3. 申请gitpage

在网上搜索也是一大堆, 重点记住的是git repo的名字要和github的名字一致.

3. 在hexo中配置gitpage

直接把配置文件拿出来了, 参考的时候照葫芦画瓢吧!

安装目录插件.

实现这个功能需要安装插件.

1. 找到hexo-renderer-markdown-it-plus插件

在hexo插件查找页面找到插件: hexo-renderer-markdown-it-plus. 这个插件是fork于hexo-renderer-markdown-it的.

2. 安装这个插件

需要将原来hexo自带的renderer替换掉.

3. 在文章中使用

在文章中以@[toc]形式使用, 但是我发现个现象就是标题1是不会显示的. 最目录里显示的最大的标题是标题2!

4. 不只是目录

错误

安装的过程中遇到了一点问题.

1. 执行 hexo init 的时候找不到一些module.

我干脆进到全局的node modules文件夹中, 找到hexo的node_modules, 把里面的都删掉又重新安装了一下就好了.

2. 执行 hexo deploy 的时候失败.

上网找, 发现是少了个组件.
在我的hexo的home目录执行命令npm install hexo-deployer-git --save

3. 执行了 hexo deploy 操作后, github.io页面没有更新.

执行了 hexo deploy 操作后, github.io页面没有更新, 但是查看GitHub的提交记录, 发现确实正常提交到仓库中了. 另外, 在本地执行hexo s 是可以正常显示的. 那么可以确定hexo是没有什么问题, 问题出在了GitHub端. 进入仓库的设置, 发现gitpage的仓库是不可以配置成私有的, 除非升级到PRO, 这个应该就是问题的成因了, 在这个页面把仓库转成公共可见即可.
